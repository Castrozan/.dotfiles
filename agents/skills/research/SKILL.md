---
name: research
description: Deep web research and synthesis for current topics, tools, vendors, or decisions. Use when user asks to research, compare, evaluate, or find the latest external information.
---

<question_definition>
Define the decision or question before searching. If the ask is broad, narrow it to the comparison, recommendation, timeline, or current-state question the user actually needs answered.
</question_definition>

<source_strategy>
Use web-enabled search when external facts may be current. Prefer primary sources first: official documentation, vendor announcements, maintainers, papers, standards, and direct repositories. Add independent secondary sources only to validate adoption, pricing, sentiment, or operational tradeoffs.
</source_strategy>

<research_workflow>
Search with 2-4 varied queries, then deepen only on the most credible results. Capture exact links, concrete dates, and the specific claim each source supports. When information is unstable or high-stakes, verify across multiple recent sources before concluding.
</research_workflow>

<synthesis>
Do not dump raw search results. Produce a direct answer, the decisive evidence, the main tradeoffs, and a recommendation when the user is making a choice. Use absolute dates, state uncertainty plainly, and separate sourced facts from your inference.
</synthesis>

<deliverable_shape>
Default shape: TL;DR, key findings, recommendation, sources. Expand into a comparison matrix or deeper report only when the user asks for depth or the decision justifies it.
</deliverable_shape>
